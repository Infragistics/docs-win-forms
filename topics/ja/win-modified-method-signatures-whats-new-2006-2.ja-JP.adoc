= 変更されたメソッド シグネチャ

{ProductName} 2006 Volume 2 では、アプリケーション スタイリングをサポートするためにコントロールに複数の改善が行われました。これらの改善に含まれるものは、いくつかのメソッド シグネチャの変更です。以下は、本リリースで修正されたメソッドのリストです。

== Infragistics.Win.UltraWinTabs.ICustomMdiTab

[options="header", cols="a,a"]
|====
|メンバ|変更

|ResolveAppearance
|このメソッドのシグネチャは、解決の順番中に使用される UIRole および ResolutionOrder を渡すように変更されました。これは必須なので DockManager は、TabbedMdiManager からの適切なロールと解決の順番を使用できます。

|====

== Infragistics.Win.UltraWinToolbarsManager.ToolBase

[options="header", cols="a,a"]
|====
|メンバ|変更

|ResolveAppearancePhase
|この保護された仮想メソッドのシグネチャは変更されました。アプリケーション スタイリングをサポートするためのロール情報だけでなく古いパラメータを含む構造を渡します。

|====

== Infragistics.Win.IValueList

[options="header", cols="a,a"]
|====
|メンバ|変更

|ResolveItemAppearance
|このメソッドのシグネチャは変更されました。アプリケーション スタイリングをサポートするために ComponentRole を渡すことが必要です。

|====
