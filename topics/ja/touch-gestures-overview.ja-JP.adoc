== はじめに

=== タッチ ジェスチャの概要

Windows 7 以前の Windows Forms コントロールがマネージされたコードでタッチ入力デジタイザーによってネイティブにユーザー操作のジェスチャをサポートしませんでした。定義済みの Windows ジェスチャのサポートを提供するために、アンマネージ コードで Windows 7 にサポートされる link:http://msdn.microsoft.com/en-us/library/ee220935.aspx[WM_GESTURE] メッセージを使用できます。

link:http://msdn.microsoft.com/ja-jp/library/windows/desktop/dd371570(v=vs.85).aspx[慣性]を持つ  _キネティック スクロール操作_   ( _タッチ パンニング_  も呼ばれる操作) は、ユーザーがタッチ画面に指をドラッグするときに発生します。移動のスピードおよび方向はウィンドウのスクロールのスピードと方向を決定します。ユーザーが指を離れた後にスクロール操作は続行します。徐々に減速して中止します。この動作は _パンニング慣性_  と呼ばれています。この減速でユーザー インタラクションはすぐに慣性パンニングを中止します。この動作は iPhone、iPad、および Windows タッチ対応の Surface デバイスなどのスマートフォンとタブレット デバイスによく使用されます。

パンニング ジェスチャもドラッグ選択のために使用できます。ユーザーが画面に _なげなわ_  をドラッグして、 _なげなわ_  矩形内の項目を選択します。

他のジェスチャにもデフォルトの動作が関連付けられています。たとえば、`長押し`ジェスチャはコンテキスト メニューを表示します。ユーザーがタッチ画面に指を離れなくて移動せずに押すジェスチャをすると、コンテキスト メニューが表示されます。

== ジェスチャをサポートするコントロール

=== コントロール

*  _UltraCalendarCombo_  
*  _UltraCombo_  
*  _UltraDayView_  
*  _UltraDesktopAlert_  
*  _UltraDockManager_  
*  _UltraExplorerBar_  
*  _UltraGanttView_  
*  _UltraGrid_  
*  _UltraListView_  
*  _UltraLiveTileView_  
*  _UltraMonthViewMulti_  
*  _UltraMonthViewSingle_  
*  _UltraMessageBox_  
*  _UltraRadialMenu_  
*  _UltraTimelineView_  
*  _UltraTree_  
*  _UltraToolbarsManager(Ribbon)_  
*  _UltraWeekView_  

_Infragistics_   コントロールは、Windows オペレーティング システムの標準ユーザー インターフェイス動作をエミュレートします。このルールを使用して、コントロールでタッチ ジェスチャ処理を実装します。

== パンニング ジェスチャ

[[_Ref349045950]]

=== スロー垂直パンニング

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーの指がタッチ画面を押して上に _遅く_ 移動した後、タッチ画面から離れるジェスチャです。
|ユーザーが指をタッチ画面で移動すると、指が項目の高さの距離を移動したときに、コントロールのスクロール位置が項目によって増加します。スクロールバーのつまみ要素が下へ移動し、ビューポートが上に移動します。表示可能領域の下にあった項目がビューに移動します。ユーザーが指を離すと、コントロールのスクロールが中止されます。

|====

[[_Ref349131123]]

=== ファスト垂直パンニング

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーが指でタッチ画面を押し、上へ _すばやく_ 移動した後、タッチ画面から離すジェスチャです。
|<<_Ref349045950,スロー垂直パンニング>>の動作とほとんど同じですが、ユーザーが指を離した後にスクロール操作が続行され、徐々に減速して止まります。

|====

[[_Ref349046163]]

=== 複数方向のパン スクロール

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーが指でタッチ画面を押して上へ中程度のスピードで移動し、1 インチ程上へ移動した後に、 _すばやく左へ移動して_ 、指をタッチ画面から離します。
|<<_Ref349045950,スロー垂直パンニング>>と同じように、コントロールが最初にスクロールしますが、スピードはより速くなります。ユーザーがスワイプの方向を垂直から水平に変更すると、コントロールは水平方向へスクロールを開始します。つまみは右へ移動し、ビューポートは左へ移動します。ユーザーが指を離した後もコントロールの水平スクロール操作は続行し、徐々に減速して止まります。

|====

[[_Ref349131139]]

=== 複数方向のパン選択

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーが指でタッチ画面を押して _左へゆっくり移動して_ 、1 インチ程左へ移動した後に、タッチ画面でコントロールの境界線内に移動します。
|最初の水平移動の後に、 _なげなわ_ と呼ばれる半透明なドラッグ矩形が表示されます。コーナーは最初の押し位置と指の現在位置です。矩形のサイズを変更すると、ドラッグ矩形と交差する項目を選択します。また、以前交差していた項目で現在交差していない項目の選択を解除します。

|====

[[_Ref349131146]]

=== 複数方向のパン展開選択

.注:
[NOTE]
====
以下のユース ケースは、操作を開始する前にコントロールで 1 項目以上選択する必要があります。
====

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|Control キーを押す場合、ユーザーが指でタッチ画面を押して _左に遅く移動し_ 、1 インチ程左へ移動した後に、タッチ画面でコントロールの境界線内に移動します。
|<<_Ref349046163,複数方向のパン選択>>ユース ケースとほとんど同じ動作ですが、以下の違いがあります。 

_なげなわ_ がドラッグ選択操作が開始した前に選択された項目と交差すると、その項目の選択を解除します。 _なげなわ_ がその項目と交差しない場合、項目が再選択されます。 

.注: 

[NOTE] 

==== 

ドラッグ選択操作に Control キーを解放すると、以前選択した項目は通常の選択ルールに従って動作します。 

====

|====

== 押してタップ ジェスチャ

[[_Ref349131162]]

=== 長押し

.注:
[NOTE]
====
以下のユース ケースは、コントロールがコンテキスト メニューを持つ必要があります。
====

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーが指でタッチ画面を押しながら _2 秒程移動_ しない場合、タッチ画面から指を離します。
|押してから 2 秒後にコンタクト ポイントにグレーの丸いインジケーターが表示されます。ユーザーが指を離したとき、コントロールのコンテキスト メニューが表示されます。

|====

[[_Ref349131170]]

=== プレス アンド タップ

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーが指でタッチ画面を押しながら _移動しない_ 場合、人差し指で任意の場所をタップし、両方の指を離します。
|2 本の指をタッチ画面から離したときに、コントロールのコンテキスト メニューを表示します。

|====

[[_Ref349131200]]

=== 標準タップ

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーが指でタッチ画面を押して離します。たとえば、 _画面のボタンをタップします。_
|コントロールはマウスの左ボタン クリックと同じ動作をします。 

* タップ位置に選択可能な項目がある場合、項目を選択します。 

* タップ位置にボタンなどのクリック可能な要素がある場合、ボタンのクリック操作を発生します。 

|====

[[_Ref349131209]]

=== キャプチャ タップ アンド ドラッグ

[options="header", cols="a,a"]
|====
|ユーザー操作|結果

|ユーザーがマウス移動をキャプチャするユーザー インターフェイス要素を押し、指を数秒間そのままにした後にタッチ画面で指をドラッグします。
|この動作はキャプチャされたマウス ドラッグ操作と同じです。たとえば、つまみのスクロールはユーザーの指を移動します。

|====
